# ADR-001: 原型阶段技术选型

## 背景

项目处于探索阶段，使用纯 HTML/CSS/JS 原型验证产品方向。需记录当前技术状态和后续技术选型要求。

## 现状

- 前端原型：`prototype.html`（原生 HTML + CSS + JS）
- 无后端服务（mock 数据）
- 无数据库（内存状态）

## 技术约束

无论后续技术选型如何，必须满足：

| 约束项 | 说明 |
|--------|------|
| 移动端优先 | 适配 375px - 480px 屏幕 |
| 无障碍支持 | 屏幕阅读器兼容，盲人可用 |
| 触达率保障 | 消息通知需可靠 |

## 待决策项

以下技术选型需在正式开发前确认：

1. **前端框架**：React / Vue / 其他
2. **后端架构**：自建 / BaaS / 已有服务
3. **用户认证**：手机号 + 验证码（已有原型流程）
4. **部署平台**：待定

## 决策

原型阶段使用原生 HTML + CSS + JS，不引入框架依赖。正式开发时按需选择。

## 理由

- 原型目的是验证产品流程，框架会增加迭代成本
- 原型可独立运行，便于分享和演示
- 技术选型应基于实际需求，而非预判

## 后续行动

- [ ] 确认前端框架（建议 React 或 Vue）
- [ ] 确认后端方案（建议 BaaS 快速启动）
- [ ] 更新本 ADR 为正式技术决策
